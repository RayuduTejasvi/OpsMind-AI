# OpsMind AI — Frontend UI Features & Button Reference

This document describes the user-facing features, buttons, and the function each control performs in the web frontend.

## Overview
- App purpose: provide authenticated users a chat interface that answers SOP (standard operating procedure) questions using an internal knowledge base derived from uploaded PDFs. Admin users can upload documents and manage the knowledge base.
- Main areas: Authentication, Chat Interface, Session Sidebar, Conversation Area, Citations Panel, Admin Panel (Document Upload / Reindex / Delete), Settings & Footer.

---

## Authentication

### Login Form
- Location: top/initial screen or `/` when not signed in.
- Fields: `Email`, `Password`.
- Buttons:
  - **Login**: Submits credentials to `POST /api/auth/login`. On success stores `accessToken`/`refreshToken` in localStorage and loads chat UI. On failure shows an inline error toast.
  - **Switch to Register**: Toggles view to the registration form.
- Secondary actions:
  - **Forgot Password** (if present): navigates to password recovery flow (not implemented in scaffold).

### Register Form
- Location: accessible from login UI.
- Fields: `Email`, `Password`, `Role` (admin/employee)
- Buttons:
  - **Register**: Submits to `POST /api/auth/register`. On success logs user in (tokens returned) and redirects to chat. Duplicate-email returns 409 with message.
  - **Back to Login**: Returns to login view.

### Header / Auth Buttons (when signed in)
- **Logout**: Clears tokens from storage and returns to login screen.
- **Profile / Me**: (Optional) Opens a small dialog showing current user details retrieved from `GET /api/auth/me`.

## Chat Interface

### Main Controls
- **New Session** (button): Creates a fresh chat session in the session sidebar. Clears the conversation area and starts a new session id. Stores session meta in user data.
- **Session Selector / Sidebar**: Lists previous chat sessions. Clicking a session loads that conversation from localStorage or backend (if saved) into the conversation pane.
- **Delete Session** (trash icon on session item): Removes a session locally and optionally calls the API to delete history: `DELETE /api/chat/history/:sessionId`.

### Conversation Area
- **Input Textbox**: Main text entry for user queries. Supports multi-line input.
- **Send / Ask** (primary button): Sends the current text to `POST /api/chat`. The request is streamed via SSE; the UI displays tokens as they arrive. On success, the assistant message is appended with citations.
- **Stop / Cancel** (secondary, appears while streaming): Stops the ongoing LLM stream and finalizes partial message.
- **Attach File** (if present): Opens admin-only upload flow; regular users cannot upload unless allowed.
- **Clear**: Clears input text.

### Message Actions (per message)
- **Copy**: Copies message text to clipboard.
- **Cite / Show Source**: Opens the cited source PDF snippet or highlights the related chunk.
- **Regenerate** (retry): Re-asks same query with same context to request a new generation.
- **Thumbs Up / Down** (feedback): Sends feedback to a metrics endpoint (not implemented by default).

## Citations Panel
- Location: Right-hand column or below message when space is limited.
- Purpose: Lists retrieved chunks and page numbers used to answer a query.
- Controls:
  - **Open PDF** (link next to each citation): Opens the stored PDF at the cited page (admin-provided viewer or download link).
  - **Copy Citation**: Copies the citation text or URL.
- Behavior: Citations are generated by the retrieval pipeline and shown with a similarity score.

---

## Admin Panel (Document Management)
Accessible only to users with `role: admin`.

### Upload Documents
- Location: Admin > Upload Documents or `/admin`.
- Controls:
  - **Choose File / Drag & Drop**: Selects PDFs (single or multi-file). File size limited by `MAX_FILE_SIZE_MB`.
  - **Upload** (primary): POSTs files via the admin route using `multipart/form-data` to `POST /api/admin/upload`. On success, the server returns a SopDocument object and begins processing (parsing + chunking + embedding). Progress is shown in a status panel.
  - **Cancel Upload**: Cancels the current file upload before it starts.
- Statuses shown for each document: `processing`, `indexed`, `error`.

### Document List
- Shows filename, upload date, page count, chunk count, and status.
- Buttons per document:
  - **Reindex**: Calls `POST /api/admin/documents/:doc_id/reindex` to regenerate embeddings.
  - **Delete**: Calls `DELETE /api/admin/documents/:doc_id` to remove document and embeddings from DB (confirm modal required).
  - **View**: Download or open stored PDF.

## Settings & Footer
- **Theme Toggle**: Switch between light/dark mode. Persists choice to localStorage.
- **API Keys / Advanced** (admin): Placeholder UI to set GEMINI_API_KEY (used if implementing real embedding/LLM).
- **Rate Limit Indicator**: Shows remaining free-tier queries (if plan enforcement is enabled).

---

## Error States & Alerts
- Global toast area displays: network errors, auth failures, file upload errors, and LLM fallback messages.
- Common messages:
  - Unauthorized → prompts to login again.
  - Email already registered → shown on register flow.
  - Upload failed → shows error details and link to retry.
  - No context found → assistant should respond with fallback like "I don't know".

---

## API Mapping (buttons → endpoints)
- `Login` → `POST /api/auth/login`
- `Register` → `POST /api/auth/register`
- `Logout` → client: clear tokens + optional server revoke
- `Get current user` → `GET /api/auth/me` (used when loading UI)
- `Send query` (Ask) → `POST /api/chat` (SSE stream)
- `Upload PDF` → `POST /api/admin/upload` (multipart)
- `List documents` → `GET /api/admin/documents`
- `Delete document` → `DELETE /api/admin/documents/:doc_id`
- `Reindex document` → `POST /api/admin/documents/:doc_id/reindex`

## Keyboard Shortcuts (recommended)
- `Enter` → Send message (within single-line mode)
- `Shift+Enter` → New line in message box
- `Ctrl+K` → New session
- `Ctrl+L` → Focus message input
- `Esc` → Close modals

## Developer Notes
- UI state persists sessions locally; server stores canonical chat history per user when saving is implemented.
- Streaming: message tokens are delivered by SSE and appended in the UI. The `Stop` button should abort the SSE connection.
- Security: All protected endpoints require `Authorization: Bearer <accessToken>` header.
